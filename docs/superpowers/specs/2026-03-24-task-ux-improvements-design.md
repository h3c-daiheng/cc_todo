# 任务管理系统用户体验优化设计

**日期**: 2026-03-24
**目标**: 提升团队协作场景下的任务查看和操作体验

## 背景

当前系统是一个功能完整的任务管理系统，包含看板视图、仪表盘、甘特图、团队协作等功能。主要使用场景是团队协作，但在任务查看组织和操作流程上存在体验问题：

- 缺少搜索功能，难以快速定位任务
- 只有看板视图，缺少列表和日历视图
- 创建任务需要打开弹窗，流程不够快捷

## 设计目标

在现有架构上渐进式增强，添加三个核心功能（优先级相同）：

1. **快速创建任务** - 减少创建任务的步骤
2. **搜索功能** - 快速定位任务
3. **视图切换** - 支持看板、列表、日历三种视图

## 详细设计

### 1. 快速创建任务

**位置**: `MyTasks.vue` 和 `TeamTasks.vue` 页面，在筛选器和看板之间

**UI 组件**:
- 内联输入框，占位符"快速添加任务..."
- 右侧"详细"按钮（可选）

**交互流程**:
1. 用户在输入框输入任务标题
2. 按回车键 → 创建任务（默认 priority=medium, status=pending）
3. 输入框清空，新任务出现在列表中
4. 如需设置更多字段，点击"详细"按钮打开完整表单弹窗

**实现要点**:
- 复用现有 `POST /api/v1/tasks` 接口
- 最小化 payload：只传 `title`，其他字段使用后端默认值
- `TeamTasks.vue` 中需要传递 `team_id`（从路由参数获取）
- 创建成功后调用 `loadTasks()` 刷新列表

### 2. 搜索功能

**位置**: 页面顶部工具栏，筛选器旁边

**UI 组件**:
- 搜索输入框，占位符"搜索任务..."
- 清除按钮（输入内容时显示）

**搜索范围**:
- 任务标题（`task.title`）
- 任务描述（`task.description`）

**交互流程**:
1. 用户输入关键词
2. 实时过滤任务列表（前端过滤）
3. 搜索和筛选器可组合使用
4. 清空搜索框恢复完整列表

**实现要点**:
- 使用 `computed` 属性过滤 `tasks` 数组
- 不区分大小写匹配
- 不需要后端 API 改动（当前数据量适合前端过滤）

### 3. 视图切换

**位置**: 页面左上角，标题旁边

**UI 组件**:
- 按钮组：看板、列表、日历三个图标按钮
- 当前视图高亮显示

**视图类型**:

#### 3.1 看板视图（已有）
- 保持现有 `TaskBoard.vue` 组件
- 三列：待处理、进行中、已完成
- 支持拖拽改变状态

#### 3.2 列表视图（新增）
- 表格形式展示任务
- 列：标题、状态、优先级、负责人、截止日期、操作
- 状态列支持下拉选择直接修改
- 点击行进入任务详情
- 操作列包含：查看详情按钮

#### 3.3 日历视图（新增）
- 月视图展示任务（按截止日期 `due_date`）
- 每个日期格子显示当天到期的任务
- 点击任务进入详情
- 没有截止日期的任务显示在侧边栏"未安排"区域

**交互流程**:
1. 用户点击视图按钮切换视图
2. 所有视图共享同一数据源（`tasks` 数组）
3. 搜索和筛选状态在视图切换时保持
4. 用户的视图偏好保存在 `localStorage`

**实现要点**:
- 创建新组件：`TaskList.vue`（列表视图）、`TaskCalendar.vue`（日历视图）
- 使用 `v-if` 根据当前视图显示对应组件
- 视图状态存储：`localStorage.getItem('taskView')` 默认 `'board'`

## 技术实现

### 前端改动

**新增组件**:
- `QuickTaskInput.vue` - 快速创建输入框
- `TaskList.vue` - 列表视图
- `TaskCalendar.vue` - 日历视图

**修改文件**:
- `MyTasks.vue` - 集成快速创建、搜索、视图切换
- `TeamTasks.vue` - 同上

**依赖**:
- 日历视图使用自己实现的简单月视图（避免引入重量级库）

### 后端改动

**无需改动** - 所有功能使用现有 API：
- `GET /api/v1/tasks` - 获取任务列表
- `POST /api/v1/tasks` - 创建任务
- `PATCH /api/v1/tasks/{id}/status` - 更新状态

## 实现顺序

建议按以下顺序实现（每个功能独立上线）：

1. **快速创建任务** - 最简单，立即见效
2. **搜索功能** - 实现简单，价值高
3. **列表视图** - 复用现有数据，实现难度中等
4. **日历视图** - 需要日历组件，实现难度较高

## 非功能需求

- **性能**: 前端过滤在任务数 < 1000 时性能可接受，超过后考虑后端搜索
- **响应式**: 所有新组件需适配移动端
- **可访问性**: 搜索框和视图切换按钮需支持键盘导航
- **测试**: 为新组件添加单元测试

## 未来扩展

如果需要进一步优化，可以考虑：

- 后端全文搜索（使用 SQLite FTS5）
- 高级筛选（多条件组合）
- 自定义视图（用户保存筛选条件）
- 命令面板（Cmd/Ctrl+K 快捷操作）
